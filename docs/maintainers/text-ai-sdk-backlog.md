# Text AI SDK 后续任务清单

## 优先级顺序

### P1. OpenAI-compatible 传输层定制

**为什么最优先**

这项能力对未来兼容更多 provider 和网关的收益最大，同时又不会破坏纯文本定位。

**范围**

- 自定义 endpoint path
- 额外 query 参数
- 额外请求头

**价值**

- 更容易接入 OpenAI-compatible 网关
- 更好支持 provider 特定路由细节
- 为后续更广的 OpenAI 协议兼容打基础

### P2. 流式生命周期监听器

**范围**

- 保留 `DeltaCallback`
- 增加可选的 richer listener，支持：
  - 开始
  - 增量文本
  - 完成
  - 错误
  - usage

**价值**

- 更适合 GUI 集成
- 更利于流式过程观测
- 为以后支持更丰富的流式元数据打基础

### P3. 响应元数据增强

**范围**

- 暴露 response ID
- 暴露模型名
- 暴露原始响应头或 provider 元数据

**价值**

- 更利于排障
- 更便于审计
- 更清楚不同 provider 的兼容行为

### P4. 重试与韧性策略

**范围**

- 对瞬时错误提供可配置重试
- 明确重试边界
- 对 `429` 和 `5xx` 类错误做显式策略

**价值**

- 更适合生产场景
- 减少临时错误直接泄露给调用方

### P5. 兼容性示例套件

**范围**

- provider 示例
- gateway 示例
- 更多请求/响应变体说明

**价值**

- 更容易上手
- 更容易理解 OpenAI-compatible 接入方式
- 降低集成成本
